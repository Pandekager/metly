## Key Design Decisions

1. **Build on Phase 21 foundation**: Use the same API structure and authentication patterns
2. **Extend beyond simple counts**: Phase 21 tracks refund_count/refund_pct; Phase 22 provides drill-down into WHY and WHICH
3. **Return reasons**: Group by common return reasons (defective, wrong size, changed mind, etc.)
4. **Product-level analysis**: Identify top products by return count and return rate
5. **Time-to-return**: Calculate average days between order and return

## API Response Shape

```typescript
interface RefundReturnResponse {
  // Summary metrics
  total_refunds: number;
  total_refund_revenue: number;
  refund_rate: number;
  avg_days_to_refund: number;
  
  // Product-level analysis
  top_refunded_products: Array<{
    product_id: string;
    product_name: string;
    refund_count: number;
    refund_rate: number;
    total_revenue: number;
  }>;
  
  // Return reasons breakdown
  return_reasons: Array<{
    reason: string;
    count: number;
    percentage: number;
  }>;
  
  // Monthly trends
  refunds_by_month: Array<{
    month: string;
    refund_count: number;
    refund_revenue: number;
    refund_rate: number;
  }>;
}
```
